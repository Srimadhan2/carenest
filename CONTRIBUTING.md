# Contributing to CareNest

## Branch Naming

- `feature/phase-N-description` — new features
- `fix/description` — bug fixes
- `docs/description` — documentation
- `chore/description` — tooling

## Commit Messages

```
type(scope): short description
```

Types: `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`

## Development Workflow

1. One phase at a time
2. One file at a time when possible
3. Run `npm run lint` and `npm run build` before committing (tests added in Phase 8.5)
4. Run `npm run format` for consistent formatting
